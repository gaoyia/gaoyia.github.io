# 文章分类取列表(article)

# /api/article/getArticle

# 获取文章分类API

注意这里是'文章分类'不是'导航菜单'
因为已经有导航菜单,感觉获取文章分类接口并没有使用的必要，只是可以方便的查询所有的文章分类，且不支持后端排序，预留一个接口，文档略

# /api/Article/getArticleList

返回文章列表即接口

# 参数

参数名 参数类型(默认值) 说明 是否可选
id +Int(1) 获取的文章类型id 是
page +Int(1) 页码 是
offset +Int(24) 每页条数 是

一页显示24（2×3×4）条，怎么排版都好看

# /api/Article/getArticleListArr

一次性获取多个分类下的多篇文章，通常用于页面首次加载，减少请求次数
【例如，首页显示多个分类的多篇文章，点更多再进入不同的文章分类，或追加列表元素到最后】

# 参数

参数名 参数类型(默认值) 说明 是否可选
ids +Array 获取的文章分类id的数组 否
page +Int(1) 页码 是
offset +Int(24) 每页条数 是

一页显示24（2×3×4）条，怎么排版都好看

# 接口返回示例

{
    "code": 200,
    "data": {
        "list": [
            {
                "id": 1,
                "name": "娱乐",
                "pid": 0,
                "article_list": [...{}]
            },
            {
                "id": 2,
                "name": "体育",
                "pid": 0,
                "article_list": [...{}]
            }
        ]
    },
    "msg": "获取成功"
}
