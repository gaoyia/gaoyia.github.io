# 文章详情(article_list)

有人会问，为什么文章分类列表不用article_type，每条文章不用article反而用article_list命名呢？  
这里做一个说明，article中保存的是文章分类，Article_list中保存的是列表中的一篇篇的文章，这个表是文章的集合，所以命名+list。

# /api/article_list/getOneAndComment

# 获取文章详情和评论

这里不会返回文章列表中请求过的信息，在SPA中应该把该接口请求的数据挂到共享数据中如果没有，再单独获取文章数据且不包含评论数据（初步这么规划，如果后期实现比较复杂再改用常规方法）

# 参数

参数名 参数类型(默认值) 说明 是否可选
id +Int(1) 获取的导航类型id 是
page +Int(1) 评论的页码 是
offset +Int(10) 每页的数量 是
content Bool(true) 是否获取正文 是
